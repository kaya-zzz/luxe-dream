# Luxe Dream（奢梦）— 设计文档

> 一个让你假装千万资产拥有者的治愈系奢侈品浏览微信小程序

## 1. 产品定位

**核心体验：** 角色扮演千万富翁，在极简美学界面中滑动浏览真实奢侈品，拥有虚拟银行账户看资产"增长"，可以将心仪商品"购买"。

**目标用户：** 想要放松解压、享受"云逛街"乐趣的年轻人。

**关键定义：**
- 纯娱乐/治愈用途，不涉及任何真实交易
- 商品信息（名称、图片、描述）基于真实产品，但价格不获取实时电商价格
- 不涉及任何商业引导、跳转购买链接或广告

## 2. 视觉风格

- **极简纯白**：Apple 式大留白，纯白/浅灰背景
- **无卡片容器**：商品抠图 PNG 直接漂浮在背景上，带轻微 drop-shadow 增加悬浮感
- **字体**：系统默认字体，品牌名用小字母间距大的形式
- **配色**：黑白灰为主，绿色表示收入，红色表示支出
- **动效**：spring 弹性缓动，所有过渡丝滑流畅

## 3. 页面结构

### 3.1 底部 Tab Bar（3 个 Tab）

| Tab | 图标 | 名称 |
|-----|------|------|
| 1 | 浏览 icon | 浏览 |
| 2 | 购物车 icon | 购物车 |
| 3 | 银行 icon | 银行 |

### 3.2 浏览页（首页）

**布局：**
- 顶部：品牌名（极淡小字，letter-spacing 大）
- 中央主体：商品抠图（占屏幕 60-70%）
- 底部：商品名称 + 简短描述 + 价格
- 最底部：加入购物车按钮（圆形图标）

**交互手势：**
- 左右滑：切换同品类下的不同商品（带弹性回弹动画）
- 上下滑：切换不同品类（带转场动画）

**品类列表（8 大类）：**
1. 包（Bags）
2. 手表（Watches）
3. 首饰（Jewelry）
4. 鞋履（Shoes）
5. 服装（Fashion）
6. 香水（Perfume）
7. 跑车（Supercars）
8. 彩妆（Beauty）

**品类切换转场：** 上滑/下滑时有过渡动画，表明进入了新品类。

### 3.3 购物车页

**布局：**
- 顶部标题："购物车"
- 商品列表：每项显示缩略图 + 商品名 + 品牌 + 价格
- 可左滑删除单个商品
- 底部固定按钮："虚拟购买"

**虚拟购买流程：**
1. 用户点击"虚拟购买"
2. 播放购买成功动效
3. 从银行账户中扣除相应金额
4. 商品从购物车移除
5. 银行页消费记录新增一条

### 3.4 银行页

**布局：**
- 顶部标题："我的银行"
- 总资产数字（大字号，monospace 字体，实时滚动动画）
- 实时利息速率（绿色，显示每秒收益）
- **本次浏览收益卡片**（绿色渐变独立卡片）：
  - 脉冲圆点 + "本次浏览收益"标题 + 浏览时长
  - 收益金额（带计数动画）
  - 收益速率
  - 里程碑进度条（¥100 → ¥500 → ¥1,000 → ... → ¥1,000,000）
  - 每 15 秒轮播趣味文案（如"坐着不动也能赚钱，真香 ✨"）
- 消费记录列表（收入绿色/支出红色）

**虚拟金融逻辑：**
- 初始资产：¥100,000,000（一亿）
- 年化利率：10%（每秒更新一次余额）
- 利息计算：`每秒利息 = 当前余额 × 10% ÷ 365 ÷ 24 ÷ 3600`
- 消费扣款：从余额中实时扣除
- 数据持久化：本地存储（wx.setStorageSync）

**Session 收益机制：**
- Session 锚点（起始时间 + 起始余额）存储在 `app.js` 的 `globalData` 中
- 整个小程序使用期间持续累加，切换 Tab 不归零
- 仅在下次冷启动时重置 session
- 定时器内纯内存计算：每秒只用内存里的锚点变量（起始时间、起始余额）按公式 `startBalance + perSecond * elapsed` 直接算出当前值，全程不读写 storage，避免磁盘 IO 卡顿和累积误差导致的数字跳动

## 5. 技术架构

### 5.1 技术栈
- **框架：** 微信小程序原生（WXML + WXSS + JS）
- **状态管理：** app.js `globalData` 管理全局 session 状态
- **数据存储：** wx.setStorageSync / wx.getStorageSync（持久化余额与消费记录）
- **动画：** CSS transition + keyframes 动画（脉冲圆点、进度条过渡）+ WXS 响应式动画
- **金额格式化：** toFixed(2) + 正则千分位（替代不稳定的 toLocaleString）

### 5.3 关键实现细节

**滑动手势（浏览页）：**
- 使用 `touchstart` / `touchmove` / `touchend` 事件
- 水平位移 > 垂直位移 → 左右滑（切换商品）
- 垂直位移 > 水平位移 → 上下滑（切换品类）
- 使用 WXS 响应式动画确保 60fps

**收益实时更新（银行页）：**
- 页面 onShow 时启动定时器（setInterval 1秒），onHide 时清除
- 每秒基于 Session 锚点纯内存计算（不逐秒累加、不读写缓存，直接按公式重算，数字平滑无抖动）：`当前资产 = sessionStartBalance + 每秒利息 × 累计秒数`
- 每秒利息 = `sessionStartBalance × 10% ÷ (365×24×3600)`
- 金额统一经 `formatAmount`（`toFixed(2)` + 正则千分位）格式化后渲染，避免抖动

**游戏化激励（银行页）：**
- 里程碑进度条：本次收益向下一档目标（100/500/1000/…）逼近，展示百分比与缺口
- 趣味文案轮播：每 15 秒切换一条鼓励文案，营造被动收入的正反馈
- 浏览时长（分:秒）与「正在赚钱」状态实时展示

**数据一致性：**
- Session 期间切换 Tab 不重置，仅冷启动重置锚点
- onHide 时将当前资产写回 storage（`balance` + `lastUpdateTime`）
- 购物车购买时先 `updateInterest` 补算利息再校验余额，同步更新 balance 与 records

## 6. 非功能性要求

- **性能：** 滑动必须 60fps，无卡顿
- **存储：** 本地存储不超过 10MB 限制
- **商品图片：** 使用懒加载，当前屏 ± 1 张预加载
- **无账号体系：** 所有数据存储在本地，换设备数据丢失（可接受）
- **无网络依赖：** 纯离线可用（商品数据打包在小程序内）
